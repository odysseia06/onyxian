# music

The musician's workspace, generalized from the maintainer's tree (KICKSTART.md §5.4): five practice/creation areas plus a knowledge shelf and copy-per-piece projects (`Projects/_Project-Template`, the academic copy-per-X pattern). Eleven templates cover the note types; the source vault's rule survives: loose ideas live in Composition first, a piece earns a project folder only when it gets serious.

Key types: `practice-log` (`logged`; drives `Practice/Practice-Log.base`), `practice-routine`, `music-exercise`, `theory-note`, `composition-idea` (`sketch`), `song-note` (`draft`), `lyric-note`, `production-note`, `listening-note` (`logged`), `music-review`, `music-knowledge`, and the five seeded `music-project-*` notes.

Variables: `root` (default `Music`). Seeded: `00 Dashboard.md`, `Goals.md`, and the project-template subtree.
