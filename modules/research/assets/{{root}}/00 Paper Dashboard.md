# Paper Dashboard

The pipeline runs on metadata, not folders: papers live flat in Paper-Summaries and Paper-PDFs, named `CitationKey - Short Title`, and every view below is the [[Paper Library.base|Paper Library]] filtering on frontmatter. No deep subfolders — use `topics`, `type`, and `status` instead.

## Working the pipeline

- New paper: PDF into Paper-PDFs, summary from the matching typed template (attack, construction, engineering, foundations, framework, protocol, survey) or the interactive Paper Summary template.
- Statuses move `to-read` → `reading` → `summarized` (→ `revisiting` when you come back); set `date_summarized` when a summary is done.
- Topic notes collect threads across papers; reading lists are ordered wikilink runs for a purpose; open questions get their own notes.
